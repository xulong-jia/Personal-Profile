## 🚀 Featured Projects

### 🎯 [CareerAgent](https://github.com/xulong-jia/careeragent)

> Local-first AI career preparation workbench.

Built with FastAPI, React, TypeScript, SQLAlchemy, Alembic, SQLite, a PostgreSQL-compatible deployment shape, RAG, and deterministic agent workflows.

- Connects resume upload / parse / versioning, JD profiling, match scoring, project rewrite suggestions, interview practice, study plans, and application tracking.
- Includes RAG knowledge base workflows, grounded answer runs, deterministic agent runs, evaluation records, Bad Case review, and privacy controls.
- Designed around reviewable outputs rather than hidden prompt state or automatic resume changes.
- Boundary: production-readiness is in progress; it is not a live SaaS, not an automatic job application sender, and the default AI path is deterministic/local.

### 🧾 [FinancialAuditAI](https://github.com/xulong-jia/FinancialAuditAI)

> Evidence-first financial document audit and procurement walkthrough platform.

Built with FastAPI, React, TypeScript, Ant Design, PostgreSQL, pgvector, Alembic, OCR provider abstraction, RAG, and a deterministic rule engine.

- Covers task creation, document upload, OCR, document classification, field extraction, procurement walkthrough rules, RAG evidence retrieval, and report generation.
- Provides human review, field correction history, Review Center, Report Center, Bad Case tracking, Evaluation Center, RBAC/admin, and provider readiness boundaries.
- Keeps source evidence, citations, rule results, and review decisions visible for audit-style inspection.
- Boundary: non-production public/synthetic acceptance only; it is not a production audit system and does not provide professional audit, legal, compliance, investment, or regulatory advice.

### 🚦 [SmartTraffic](https://github.com/xulong-jia/SmartTraffic)

> Local smart traffic event detection and analysis platform.

Built with FastAPI, React, TypeScript, YOLOv8, a DeepSORT adapter, trajectory engine, SQLite, and Docker Compose.

- Supports video upload, analysis runs, YOLOv8 detection/tracking wrappers, trajectory semantics, configurable zones, event rules, alerts, manual review, Bad Cases, local evaluation, and PDF/JSON/CSV report export.
- Final local run generated 14 danger-zone intrusion events, 14 alerts, 1 bad case, 2 zone statistic records, and local event metrics.
- Event detection is rule-based over tracked trajectories, zones, and configured rules, not a dedicated traffic-law model.
- Boundary: local validation prototype; not law-enforcement-grade, not production-ready, not an official COCO / TrackEval benchmark, and pixel speed is not calibrated real-world speed.

### 🚗 [YOLOv8 Vehicle and Pedestrian Detection System](https://github.com/xulong-jia/yolov8-vehicle-pedestrian-detection)

> Vehicle and pedestrian detection, tracking, local demo, API serving, and Docker delivery.

Built with YOLOv8 / Ultralytics, FastAPI, React, Streamlit, Docker, tracking artifacts, and CSV/JSON outputs for local detection, tracking, lightweight video analytics, API serving, and Docker delivery.

- Covers dataset validation, YOLOv8n / YOLOv8s / YOLOv8m experiments, image inference, video inference, tracking artifacts, React dashboard, Streamlit demo, FastAPI API, and Docker smoke checks.
- Recommended model: YOLOv8s. Demo image detection returned Truck and mini-truck boxes.
- Demo video task generated 21,988 detection rows and 21,988 tracking artifact rows for local inspection.
- Boundary: local technical validation / course project; not a production deployment, not an official COCO / TrackEval result, current demo tracking uses a synthetic tracker, and current demo run does not claim effective event / ROI / line-counting results.

## 🧪 Engineering Themes

- **Local-first systems:** reproducible workflows, ignored runtime artifacts, and clear safety rules.
- **API-first backends:** typed schemas, persisted records, and runnable local flows.
- **Workbench UIs:** dashboards that expose state, evidence, outputs, and failure cases.
- **Evidence and review:** citations, field sources, traffic event evidence, and model output checks.
- **Evaluation loops:** Bad Case tracking, local metrics, regression fixtures, and report exports.
- **Honest AI boundaries:** deterministic defaults, explicit provider paths, and no unsupported production claims.
- **Docker/local delivery:** local demo paths without committing models, datasets, databases, reports, secrets, or generated outputs.
